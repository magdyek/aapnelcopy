# aaPanel Smart Migrator 🚀

أداة ذكية ومفتوحة المصدر لنقل المواقع وقواعد البيانات بين سيرفرات **aaPanel** بسهولة، وبدون الحاجة لمعرفة أو استخدام أداة الـ SSH. الأداة تعتمد بالكامل على الـ API الخاص بلوحة aaPanel لتنفيذ عمليات نقل سريعة، آمنة، ومستقرة.

## ✨ المميزات (Features)

* **نقل الموقع بالكامل أو أجزاء منه**: يمكنك النقل الشامل (موقع + قاعدة بيانات)، أو نقل الملفات فقط، أو قاعدة البيانات فقط.
* **الاستغناء الكامل عن SSH**: تعتمد الأداة على `cURL` والـ API الافتراضي للوحة aaPanel لسحب البيانات، مما يتفادى مشاكل الحماية وانقطاع الـ SSH.
* **إدارة النسخ الاحتياطية بذكاء**:
  * عرض قائمة بالنسخ الاحتياطية المتوفرة على السيرفر القديم، لاختيار إحداها بدلاً من توليد نسخة جديدة وتضييع الوقت.
  * إمكانية **حذف النسخ الاحتياطية القديمة** من السيرفر القديم بضغطة زر.
* **الفك التلقائي لقواعد البيانات (Auto-Decompression)**: تكتشف الأداة قواعد البيانات المضغوطة (`.zip` أو `.gz`) وتقوم بفك ضغطها محلياً قبل الاستيراد لضمان استعادة الجداول بشكل سليم.
* **تحديث إعدادات الاتصال (Auto-Config)**: تقوم الأداة بتحديث ملفات `.env` (لمشاريع Laravel) أو `wp-config.php` (لمشاريع WordPress) ببيانات قاعدة البيانات الجديدة تلقائياً بمجرد انتهاء النقل.
* **واجهة مستخدم عصرية (UI)**: الواجهة مبنية بـ Tailwind CSS بتصميم زجاجي داكن (Dark Glassmorphism) مع مؤشر لتتبع حالة النقل خطوة بخطوة.

## 🛠️ المتطلبات (Requirements)

1. سيرفر جديد (الوجهة) يعمل عليه aaPanel.
2. يتطلب السيرفر الجديد استضافة هذه الأداة (ترفع ملفات الأداة في فولدر فرعي داخل أي موقع يعمل بـ PHP).
3. مفتاح API (API Key) من كلا السيرفرين (القديم والجديد). (يتم إصداره من `Settings -> API` في لوحة aaPanel).
4. تفعيل وصول الـ IP الخاص بالسيرفر الجديد داخل إعدادات الـ API في السيرفر القديم.

## 🚀 طريقة الاستخدام (How to use)

1. **التحميل والرفع**: قم برفع ملفات المشروع (`index.php`, `ajax.php`, ومجلد `src/`) داخل فولدر جديد على موقعك في السيرفر الجديد (مثال: `yourdomain.com/migrator`).
2. **تشغيل الأداة**: افتح الرابط في المتصفح `http://yourdomain.com/migrator/index.php`.
3. **الاتصال**:
   - أدخل رابط السيرفر الجديد والـ API Key الخاص به، واضغط `فحص الاتصال`.
   - أدخل رابط السيرفر القديم والـ API Key الخاص به، واضغط `فحص الاتصال`.
4. **تحديد النقل**: 
   - اختر اسم الموقع والداتابيز من القوائم التي ظهرت من السيرفر القديم.
   - إذا كان لديك نسخة احتياطية سابقة جاهزة لقاعدة البيانات، يمكنك اختيارها من القائمة لتقليل وقت التجهيز.
5. **بيانات السيرفر الجديد**: أدخل اسم الدومين الجديد واسم وباسوورد قاعدة البيانات (ستقوم الأداة بإنشاء كل هذا تلقائياً على السيرفر الجديد).
6. **بدء النقل**: اضغط على زر `بدء النقل الذكي` وراقب شاشة التقدم حتى النهاية!

## ⚙️ التكوين (Structure)

* `index.php`: واجهة المستخدم (Frontend).
* `ajax.php`: المعالج الخلفي (Controller) لربط الواجهة بالمكاتب.
* `src/AAPanelAPI.php`: الكلاس المسؤول عن التخاطب مع API لوحتي التحكم (القديمة والجديدة) وإرسال أوامر الإنشاء وسحب الملفات.
